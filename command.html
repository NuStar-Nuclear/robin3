Run in Command Line¶

ROBIN3 click-based application contains various commands that allow user to run simulation, plot geometry, interact with other nuclear programs, etc. Command line format is:

robin3 COMMAND [ARGS]

robin3 run¶

Main command of robin3. Run a ROBIN3 simulation.

INPUT_FILE is the name of file with path in which the model defined, both yaml or json format are acceptable.

robin3 run [OPTIONS] INPUT_FILE

Options

--library <library>¶

The nuclear data library file name with path, only used by micro-xs cases. Nuclear data library file installed in package will be used as default.

--material-databank <material_databank>¶

The material databank file name with path, only used by micro-xs cases. Material databank file installed in sys.path will be used as default.

--assembly <assembly>¶

The name of target assembly which you want to simulate, only used when you want to perform a single assembly calculation defined in a whole core model input file.

--cyclic-track, --no-cyclic-track¶

Whether to use cyclic track, only used for single assembly cases. Cyclic track will accelerate simulation by reduce the total number of MOC iteration, but only works on CPU without OpenMP multi-threads parallelization.

Default

False

--use-gpu, --no-use-gpu¶

Whether to use GPU, GPU will significantly accelerate simulation by parallel the MOC tracking calculation. If –use-gpu but none of GPU avaiable on the platform, CPU with OpenMP multi-threads parallelization will be used.

Default

False

--device-id <device_id>¶

The device id of which GPU used, only works if –use-gpu.

Default

0

--num-thread-blocks <num_thread_blocks>¶

Total number of thread blocks used, only works if –use-gpu.

Default

128

--num-threads-per-block <num_threads_per_block>¶

Number of threads per block used, only works if –use-gpu, should be a multiple of 32.

Default

128

--number-groups-edit <number_groups_edit>¶

Number of energy groups used to edit calculation result, only works for micro-xs cases.

Default

2

Options

2|3|4|8

--output-file <output_file>¶

The name of output file with path, the model name (assembly name or core name) with current directory will be used as default.

--logger-file <logger_file>¶

The name of rotating logger file use to store logger info.

Default

/home/yuejin/Cloud/robin3/docs/robin3.log

--logger-level <logger_level>¶

The logger level used to output logger info.

Default

info

Options

debug|info|warn|error

--verbose, --no-verbose¶

Whether to print iteration info to the logger, a progress bar will be used if verbose is False, only works for single assembly micro-xs cases.

Default

True

--profile-mode, --no-profile-mode¶

Whether to turn on profile mode, if profile mode turns on, a profile file will be generated.

Default

False

--write-json-debug, --no-write-json-debug¶

Whether to write a json file for cpp debugging.

Default

True

Arguments

INPUT_FILE¶

Required argument

All the output files will be placed in a subfolder with the same name as the input file (no filename extension).

Note

If you’re using the Python API, robin3.run() is equivalent to running robin3 run from the command line.

robin3 plot-geometry¶

Plots a 2D surface of the materials in the geometry by a model input file.

The geometry and material data of the model should be defined in the input file, ROBIN3 will perform geometrical computation such as volume calculation before plotting.

FILENAME is the name of file with path in which the model defined, both yaml and json format are acceptable.

robin3 plot-geometry [OPTIONS] FILENAME

Options

--assembly <assembly>¶

The name of target assembly which you want to plot, only used when you want to plot a single assembly defined in a whole core model input file.

--show-meshes, --no-show-meshes¶

Whether to show fine meshes–flat source regions used by MOC.

Default

False

--show-edge, --no-show-edge¶

Whether to show the mesh edge.

Default

True

--save-fig, --no-save-fig¶

Whether to save the plotting figure.

Default

False

--num-process <num_process>¶

Number of process used to perform geometry computation. Note that geometry computations are parallelized in assembly level by multi-processing, which means this argument only works for core model ploting.

Default

32

Arguments

FILENAME¶

Required argument

robin3 plot-flat-source-regions¶

Plots a 2D surface of the flat source regions in the geometry by a model input file.

Flat source regions are used by method of characteristics (MOC), neutron source in a flat source region assumed to be constant. The geometry and flat source regions data of the model should be defined in the input file, ROBIN3 will perform geometrical computation such as volume calculation before plotting.

FILENAME is the name of file with path in which the model defined, both yaml and json format are acceptable.

robin3 plot-resonance-regions¶

Plots a 2D surface of the resonance regions in the geometry from a single assembly micro-xs calculation output file in HDF5 format.

Resonance regions are used by resonance calculation, unlike the geometry, ROBIN3 does not kown which material should be treated as resonance material until nuclear data library loaded, so it use output file to plot the resonance regions, ROBIN3 will perform geometrical computation such as volume calculation before plotting.

FILENAME is the name of output file in HDF5 format with path.

robin3 plot-resonance-regions [OPTIONS] FILENAME

Options

--show-edge, --no-show-edge¶
