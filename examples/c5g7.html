Defining Pin Stuctures¶

Due to the same geometry used for different pin structures in C5G7 benchmark problem, here we define a function to generate pin structure:

[12]:

def generate_pin_structure(name, material):
    return {
        "name": name,
        "primitives": [{"type": "CIRCLE", "radius": 0.54}],
        "regions": [{"material": material, "space": "1"}]
    }

Use function defined above to generate pin structures:

[13]:

pin_names = ["FC", "GT", "UO", "M4", "M7", "M8", "CR"]
materials = ["FC", "GT", "UO2", "MOX43", "MOX70", "MOX87", "CRD"]
pin_structures = [generate_pin_structure(name=n, material=m) for n, m in zip(pin_names, materials)]

Defining Core¶

Core object is used to define core model, here we define the C5G7 mini core with 4x4 assemblies, and set the symmetry to 4.

Composing a Input Model¶

The input model for a typical full-core macroscopic cross section case includes following parts: 1. cross sections; 2. structures; 3. assemblies; 4. core.

[16]:

model = {
    "cross_sections": macro_xs,
    "structures": pin_structures,
    "assemblies": [assembly_UO2, assembly_MOX],
    "core": core
}

Visualizing Input Model¶

Instantiates a robin3.plotter to visualize the model

[17]:

plotter = robin3.Plotter(model)

Display our C5G7 mini core by plot_geometry method:

[18]:

fig = plotter.plot_geometry()

Run ROBIN3¶

Running ROBIN3 from Python can be done using the robin3.run() function. This function allows you to use GPU by setting use_GPU=True if it needed, GPU will significantly reduce the running time.

[19]:

result = robin3.run(model, use_gpu=True)

[20]:

print(result)

<robin3.result_process.ResultProcessor object at 0x7f9d9d759320>

The robin3.run() function return a robin3.ResultProcessor instance. All the result data can be accessed by the result_data attribute, and the result data are saved by a HDF5 file named by result_file attribute, summary result data such as keff can be inspected by method summary which return a pandas DataFrame.

[21]:

print(result.result_file)
df = result.summary()
print(df)

Result Visualization¶

Result visualization from Python can be done using a robin3.Plotter instance generate by ResultProcessor’s method generate_plotter.

[22]:

plotter = result.generate_plotter()

Here we can plot input model as well, this time we show the fine meshes:

[24]:

fig = plotter.plot_geometry(show_meshes=True)

From the output file, many result items can be displayed by the robin3.Plotter Class. For example, we can plot the pin power distribution at any burnup points that we calculated by plot_pin_power method:

[25]:

fig = plotter.plot_pin_power()

Visualize the mesh flux distribution of each energy group:

[26]:

for g in range(7):
    fig = plotter.plot_mesh_flux(group=g+1)
